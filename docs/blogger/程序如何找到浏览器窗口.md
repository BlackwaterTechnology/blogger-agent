程序在执行 `blogger --payload articles/OpenAI的四层分层架构 --platform wechat` 命令后，寻找浏览器窗口和打开文章创建页面的过程，主要是通过 **AppleScript 自动化控制 macOS 上的 Google Chrome 浏览器** 来实现的。

以下是具体的运作机制，代码核心逻辑在 `src/blogger/core/chrome.py` 和 `src/blogger/platforms/wechat.py` 中：

### 1. 如何找到浏览器窗口及页面
在 `wechat.py` 中，程序调用了 `self.chrome.find_global_tab(["https://mp.weixin.qq.com"])`，这个方法的底层运行了一段 **AppleScript** 脚本：

*   **遍历所有 Chrome 窗口与标签页**：AppleScript 会让系统遍历当前 Google Chrome 打开的所有的 Window（窗口）以及其中的 Tab（标签页）。
*   **匹配 URL 前缀**：在遍历时，脚本会读取每个标签页的 URL，并判断是否以 `https://mp.weixin.qq.com`（微信公众平台）开头。
*   **激活并返回索引**：一旦找到符合条件的标签页，AppleScript 会将该窗口置于前台，激活该标签页（`activate`），并返回该窗口的索引（`Window Index`）和标签页的索引（`Tab Index`），供后续操作使用。
> *注：这意味着在执行命令前，你必须已经在 Chrome 浏览器中登录并打开了微信公众平台的后台页面。*

### 2. 如何打开“文章创建”页面
找到微信公众平台的标签页后，程序会检查该标签页当前的 URL，判断是否已经在文章编辑器页面：

*   **检查当前 URL**：程序判断获取到的 URL 中是否包含 `"appmsg_edit"` 字符串。
*   **提取 Token（关键步骤）**：如果当前不在编辑器页面（比如在后台首页），程序会解析当前的 URL，从中提取出查询参数 `token`（微信公众平台后台的每个会话都依赖此 Token 鉴权）。
*   **拼接新的编辑器 URL**：拿到 `token` 后，程序会拼接出直接指向“新建图文素材”的特定链接：
    `https://mp.weixin.qq.com/cgi-bin/appmsg?t=media/appmsg_edit_v2&action=edit&isNew=1&type=77&token={token}&lang=en_US`
*   **指令跳转**：最后，程序调用 `self.chrome.set_tab_url(...)`，再次通过 AppleScript 修改刚刚找到的那个标签页的 URL，强制该标签页直接加载拼接好的文章编辑器页面，并等待页面加载完成（约 15 秒）。

通过这种 **“寻找已有会话标签页 -> 提取 Token -> 强制跳转到目标页面”** 的方式，程序非常巧妙地绕过了复杂的登录验证机制。
