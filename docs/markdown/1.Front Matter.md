### 概念解析：Front Matter
图片中展示的这种位于 Markdown 文件最上方、由 `---`（三根横线）包围起来的键值对区域，被称为 **Front Matter**（中文常译为“前置元数据”或“前言”）。

从图片中的语法特征来看，这具体是一段 **YAML Front Matter**。

### Markdown 是否支持这种语法？
答案是：**原生不支持，但现代 Markdown 生态广泛支持**。

* **原生 Markdown：** 标准的 Markdown 规范（John Gruber 最初的定义）并不包含这个概念。如果你用最基础的 Markdown 编辑器打开它，它可能只会被渲染成普通的文本和一条水平分割线。
* **扩展支持：** 在现代的 Markdown 写作和发布生态中，Front Matter 已经成为了**事实上的标准**。

### 它的核心作用与工作原理
Front Matter 的主要作用是为当前 Markdown 文件提供**元数据（Metadata）**，帮助程序在渲染正文之前理解这篇文档的属性。

* **解析拦截：** 当你使用静态网站生成器（如 Hexo, Hugo, Jekyll）、现代笔记软件（如 Obsidian）或者 Docs-as-code 工具（如 MkDocs, Docusaurus 构建技术文档）读取这个文件时，程序会首先把 `---` 之间的内容拦截并解析出来。
* **数据注入：** 解析出来的属性（例如图片里的 `title`, `author`, `desc`）会被当成变量提取走。它们不会显示在最终的文章正文里，而是会被注入到页面的 HTML 模板中（例如生成网页的 `<title>`、页面的 SEO Meta 标签、作者署名栏等）。
* **正文渲染：** 只有 `---` 区域以下的内容（如图片中的 `# 标题`），才会真正交给底层的 Markdown 引擎去转换成网页正文。

### 常见的格式类型
除了图片中使用的 YAML 格式，部分工具也支持其他的 Front Matter 语法：
* **YAML（最常见）：** 由 `---` 包围，采用键值对写法。
* **TOML：** 由 `+++` 包围。
* **JSON：** 直接由 `{ }` 包围。
